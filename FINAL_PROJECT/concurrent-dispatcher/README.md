This project is a task scheduling simulator written in Rust that shows how multiple tasks can be handled at the same time using threads similar to how an operating system works. The project is organized using a standard Rust Cargo structure. The main project folder is called concurrent-dispatcher. Inside it there is a src folder which contains all of the source code files for the system. These files include main.rs which starts the program, generator.rs which creates tasks, dispatcher.rs which manages the task queue, worker.rs which runs tasks using multiple threads, monitor.rs which tracks system performance, state.rs which stores shared system data, and task.rs which defines what a task is. The project also includes a Cargo.toml file which is used by Rust to manage dependencies and build the project. This structure helps keep the code organized by separating each part of the system into its own file so it is easier to understand and maintain.To build the project you use cargo build and to run it you use cargo run, You must make sure you are in the FINAL_PROJECT folder (if not you must use cd .. command) once in the folder open the terminal and run "cargo run" the program should run from there. The program creates random tasks using a fixed seed. I experimented with different numbers of tasks including smaller runs and larger runs up to 1000 tasks to see how the system behaves under different levels of load. Each task is either a CPU task or an IO task and they arrive over time instead of all at once. A generator creates the tasks a dispatcher puts them into a shared queue and 8 worker threads take tasks from the queue and run them. There is also a monitor that checks how busy the system is while it is running. The system prints important results like how many tasks finished how long everything took average waiting time turnaround time CPU usage and how busy the workers were. The program also compares two situations a normal balanced workload and a more stressful workload to see how the system performs under pressure.
